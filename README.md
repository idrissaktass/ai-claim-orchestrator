# ClaimFlow - Dashboard Project
This project is a web interface designed to visualize and manage an insurance claims process.

## Key Implementations
### 1. Dynamic Workflow Management
The application displays all data received from a JSON file on the screen using the appropriate interface.

**Note Insertion:** Users can add their own notes between any two steps on the timeline. This change is reflected instantly without refreshing the page.

**Different Card Structures:** Each step contains its own specific information. This structure is built using the “Registry Pattern” logic, meaning it’s very easy to add a new step type.

### 2. Artificial Intelligence (AI) Integrations
**AI Assistant:** The “Explain with AI” button located below each step simulates an assistant that explains the technical terms related to that step to the user.

**Document Analysis:** An interactive workflow has been created in the “Deduction Reason” section where the AI scans and validates an uploaded document.

### 3. Design and User Experience
**Responsive Design:** The dashboard works seamlessly on both mobile and desktop. On mobile, the sidebar structure slides to the top for ease of use.

**Visual Distinction:** Steps generated by the system are visually distinguished from notes added later by the user.

## Technologies Used
**React & TypeScript:** The project’s core framework and type safety.

**Zustand:** Used for state management. 

**Tailwind CSS:** Used for styling and responsive design.

**Lucide React:** Used for icons.

## Getting Started
Follow these steps to run the project locally:

**Clone the repository:**

```bash
git clone https://github.com/idrissaktass/ai-claim-orchestrator.git
```
```bash
cd ai-claim-orchestrator
 ```

Install dependencies:

```bash
npm install
```
Start the development server:

```bash
npm run dev
```
